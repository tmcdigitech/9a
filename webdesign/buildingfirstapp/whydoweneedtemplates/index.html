Why do we need templates?

  10 minute read  

At the bottom of this page is the source code for our web app, but now it serves a typical web page when you go to http://127.0.0.1:5000/demo. Copy it, try it out, and then come back here and read on…

It’s insane, right?! You can imagine that if this was how we handled every page, our app.py file would get out of hand pretty quickly. And if we had multiple pages, most of them would have a lot in common, because all the layout and styling would look very similar from page to page (just like on the website you’re reading now); so there’d be a lot of repetition between pages. So it’d be great if we could factor out all the HTML, and keep it separate from the Python, as much as possible. And so we will introduce a template.
